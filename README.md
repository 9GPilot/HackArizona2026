# Study Project

React + Node prototype for a study platform with:

- session-based authentication
- `Dashboard`
- `AI Tutor`
- `Study Groups`
- user-specific progress

The current app opens on a login/sign-up screen first. After authentication, users enter the main application and only see their own tutor chats, progress, and group history.

## Stack

- React 18
- Vite 5
- Express
- SQLite via `better-sqlite3`
- `express-session`
- `bcryptjs`
- Plain CSS

## Run

1. Install dependencies:

```bash
npm install
```

2. Start frontend + backend together:

```bash
npm run dev
```

3. Run only the API server:

```bash
npm run server
```

4. Build the frontend:

```bash
npm run build
```

## Current Scope

- Login / sign-up before the main app
- Password hashing with `bcryptjs`
- Session auth with `express-session`
- SQLite tables for users, chats, typing activity, study groups, group actions, and user progress
- Subject cards for Chemistry, Physics, Environment, IQ, Math, and Computer Science
- API-backed tutor chat persistence
- API-backed study group creation and per-user history
- Per-user progress view shown under the logged-in username tab
- Responsive layout for desktop and mobile

## Database Tables

- `users`
- `ai_tutor_chats`
- `typing_keyboard_activity`
- `study_groups`
- `group_visits_joined_sessions`
- `user_progress`

## Design Direction

Visual goals implemented in this version:

- Neumorphic main app with light and dark themes
- Raised auth panel before entry to the app
- Soft shadows, tactile controls, rounded panels, and restrained motion

## Notes

- The tutor response is still a local placeholder generated by the API layer
- The SQLite database file is created automatically on first backend start
- For production, replace the default session secret and move sessions to a persistent store
